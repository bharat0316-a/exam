1.
What is an algorithm ?
Answer & Solution
An algorithm is a step-by-step, unambiguous, finite, and effective procedure
for solving a well-defined computational problem. It takes zero or more inputs,
produces at least one output, and terminates after a finite number of steps. In
essence, it is a recipe or a precise set of instructions to accomplish a specific task.
1. Input : Zero or more quantities are externally supplied.
2. Output : At least one quantity is produced.
3. Definiteness :Each instruction is clear and unambiguous.
4. Finiteness :If we trace out the instructions of an algorithm,then for
all cases,the algorithm terminates after a finite number of steps.
5. Effectiveness :Every instruction must be very basic so that it can be
carried out, in principle,by a person using only pencil and paper.It
is not enough that each operation be definite as in criterion 3; it also
must be feasible.
2.
Define Time and space complexity of an algorithm.
Answer & Solution
Time complexity:
A measure of how the running time of an algorithm grows with the size of its input n.
It is usually expressed using asymptotic notation (e.g., O(n),O(n2)) and counts the
number of basic operations rather than actual clock time.
Space complexity:
A measure of the total memory an algorithm uses as a function of input size n. It includes
memory for input, variables, data structures, and recursion/stack frames, and is also expressed
asymptotically (e.g., O(1), O(n)).

Answer & Solution

4.
Define DFS and BFS.
Answer & Solution
Depth-First Search (DFS):
A graph/tree traversal algorithm that explores as far along each branch as possible before backtracking.
It uses a stack (either explicit or via recursion) to remember the path of exploration.
Breadth-First Search (BFS):
A graph/tree traversal algorithm that visits all the vertices at the present depth level before moving to the
next level. It uses a queue to process nodes in increasing order of their distance from the start node.
5.
What is greedy technique ?
Answer & Solution
A problem-solving strategy in which an algorithm builds a solution step-by-step, always choosing
the best (locally optimal) option available at each step with the hope that these local choices will lead to a
globally optimal solution.
It does not reconsider previous choices, so it is simple and efficient, but it works correctly only for problems
that satisfy properties like greedy choice and optimal substructure (e.g., Kruskal‚Äôs and Prim‚Äôs algorithms).

9.
Explain binary search technique with example.
Answer & Solution
Binary Search is an efficient searching technique used on a sorted list or array.
It works by repeatedly dividing the search interval into two halves, instead of scanning every
element like linear search.
BASIC IDEA
Start with the first index as low and the last index as high.
Find the middle position: mid = (low + high) / 2.
Compare the middle element with the key (item to search):
‚Ä¢ If key == middle element ‚Üí search is successful.
‚Ä¢ If key < middle element ‚Üí search in the left half (set high = mid ‚àí 1).
‚Ä¢ If key > middle element ‚Üí search in the right half (set low = mid + 1).
Repeat steps until:
‚Ä¢ key is found, or
‚Ä¢ low becomes greater than high (means element not found).
CONDITION
The list MUST be sorted before performing binary search.
TIME COMPLEXITY
O(log n)
Because the list size is reduced to half in every step.
EXAMPLE
Suppose the sorted array is:
A = [3, 8, 15, 20, 25, 30, 45]
We want to search for key = 25
Step 1:
low = 0
high = 6
mid = (0 + 6) / 2 = 3
A[mid] = A[3] = 20
25 > 20 ‚Üí search in right half
Step 2:
low = 4
high = 6
mid = (4 + 6) / 2 = 5
A[mid] = A[5] = 30
25 < 30 ‚Üí search in left half
Step 3:
low = 4
high = 4
mid = (4 + 4) / 2 = 4
A[mid] = A[4] = 25
Key found at index 4
So, the element 25 is successfully found.
WHEN ELEMENT IS NOT FOUND
If low becomes greater than high, then:
Element does not exist in the list.

Answer & Solution
The Knapsack Problem is a fundamental optimization problem in the study of
algorithms. It deals with selecting items to place into a knapsack (bag) with a limited carrying
capacity, such that the total profit (or value) of the selected items is maximized without
exceeding the weight capacity of the knapsack.
We are given:
‚Ä¢ A knapsack of maximum capacity W
‚Ä¢ A set of n items
‚Ä¢ Each item i has
weight = wi
profit/value = pi
The objective is to select items so that:
‚Ä¢ Total weight ‚â§ W
‚Ä¢ Total profit is maximum
Types of Knapsack Problem
0/1 Knapsack Problem
In this version, each item can either be selected completely (1) or not selected at all (0).
Partial selection of items is not allowed.
This problem is generally solved using Dynamic Programming or Branch and Bound techniques.
Fractional Knapsack Problem
In this version, fractions of items can be selected.
This problem is solved using the Greedy method by selecting items in decreasing order of profit-to-weight ratio.
Example of 0/1 Knapsack Problem
Knapsack capacity:
W = 50
Items available:
Item Weight Profit
1 10 60
2 20 100
3 30 120
We check valid combinations:
Case 1:
Select item 1 and item 2

Select item 2 and item 3
Total weight = 20 + 30 = 50
Total profit = 100 + 120 = 220 ‚Üê Maximum profit
Case 3:
Select item 1 and item 3
Total weight = 10 + 30 = 40
Total profit = 60 + 120 = 180
Case 4:
Selecting all items is not allowed
Total weight = 60 > 50
Therefore, the optimal solution is to select Item 2 and Item 3.

11.
Write a short note on NP-problem.
Answer & Solution
NP stands for ‚ÄúNondeterministic Polynomial time‚Äù.
An NP problem is a decision problem for which a given solution can be verified in polynomial time by a deterministic
Turing machine.
In other words, if someone gives a proposed answer to the problem, we can check (verify) whether it is
correct in polynomial time,
even if we do not know how to find that answer efficiently.
Formally:
NP = the class of problems whose solutions are verifiable in polynomial time.
Important related classes:
‚Ä¢ P: Problems that can be solved in polynomial time.
‚Ä¢ NP: Problems whose solutions can be verified in polynomial time.
Relationship:
Every problem in P is also in NP, but it is still unknown whether P = NP.
NP-Complete problems:
These are the ‚Äúhardest‚Äù problems in NP. If any NP-Complete problem can be solved in
polynomial time, then every NP problem can be solved in polynomial time.
Examples of NP-Complete problems:
‚Ä¢ 0/1 Knapsack problem
‚Ä¢ Travelling Salesman problem (decision version)
‚Ä¢ Hamiltonian Cycle problem
‚Ä¢ Graph Coloring problem
NP-Hard problems:
These are at least as hard as NP-Complete problems, but they may not belong to NP (their solutions may not
even be verifiable in polynomial time).

12.
a) Explain worst-case,best-case and average-case efficiency.
Answer & Solution
When analysing an algorithm, we measure how much time or space it uses for different kinds of
inputs of the same size n. This leads to three types of efficiency:
Best-case efficiency
This represents the minimum time (or space) taken by the algorithm for any input of size n.
It describes the ‚Äúfastest possible‚Äù performance of the algorithm.
Example:
In linear search, if the first element is the key, the best-case time is O(1).
Worst-case efficiency
This represents the maximum time (or space) taken by the algorithm for any input of size n.
It describes the ‚Äúslowest possible‚Äù performance and guarantees an upper bound.
Example:
In linear search, if the key is not present or is the last element, the worst-case time is O(n).
Average-case efficiency
This represents the expected time (or space) taken by the algorithm over all possible inputs of size n, assuming each input is equally likely.
It gives a more realistic measure of performance.
Example:
In linear search, assuming the key is equally likely to be anywhere, the average-case time is O(n).
Conclusion:
‚Ä¢ Best-case shows the minimum effort
‚Ä¢ Worst-case gives the guaranteed upper bound
‚Ä¢ Average-case gives expected practical behaviour

b) Compare iterative method with recursive method.
Answer & Solution
Iteration and recursion are two different approaches used to repeat a set of instructions in an algorithm or program.
Difference between Iteration and Recursion
Definition
Iteration:
A process of repeatedly executing a set of statements using looping constructs such as for, while, or do-while.
Recursion:
A process in which a function calls itself directly or indirectly to solve a problem.
Termination Condition
Iteration:
Terminates when the loop condition becomes false.
Recursion:
Terminates when a base (stopping) condition is reached.
Use of Memory
Iteration:
Uses constant memory because variables are reused.
Recursion:
Uses more memory because each recursive call is stored in the function call stack.
Performance
Iteration:
Generally faster and more efficient.
Recursion:
Slower due to function call overhead and stack usage.
Simplicity of Code
Iteration:
Code may be longer but more efficient.
Recursion:
Code is often shorter and easier to understand for problems like tree traversal, but less efficient.
Risk of Error
Iteration:
No risk of stack overflow.
Recursion:
Risk of stack overflow if base condition is missing or recursion is too deep.
Example Problems
Iteration:
Used in loops, counting, searching, sorting, etc.
Recursion:
Used in problems like factorial, Fibonacci series, Tower of Hanoi, tree/graph traversal, etc.

a) Explain selection sort technique using Brute force method.
Answer & Solution
Selection Sort is a simple comparison‚Äìbased sorting technique that follows the Brute Force method.
In this method, the list is divided into two parts: the sorted part (initially empty) and the unsorted part. The algorithm
repeatedly selects the smallest element from the unsorted part and places it in its correct position in the sorted part.
Working Principle
For each position i in the list (from first to last ‚àí 1):
Assume the element at position i is the smallest.
Compare it with every element in the unsorted portion of the list.
If a smaller element is found, update the position of the smallest element.
After checking all elements, swap the smallest element with the element at position i.
Repeat the same process for the remaining unsorted elements.
This is called a Brute Force approach because it simply tries all possibilities by comparing every pair in the unsorted part
until the smallest is found.
Example
Consider the list:
25, 10, 35, 15, 5
Pass 1:
Smallest element = 5
Swap 5 with 25
List becomes: 5, 10, 35, 15, 25
Pass 2:
Remaining list: 10, 35, 15, 25
Smallest = 10
Already in correct position
List remains: 5, 10, 35, 15, 25
Pass 3:
Remaining list: 35, 15, 25
Smallest = 15
Swap 15 with 35
List becomes: 5, 10, 15, 35, 25
Pass 4:
Remaining list: 35, 25
Smallest = 25
Swap 25 with 35
List becomes: 5, 10, 15, 25, 35
Now the list is sorted.
Time Complexity
Best case: O(n¬≤)
Worst case: O(n¬≤)
Average case: O(n¬≤)
Space Complexity: O(1)
Characteristics
‚Ä¢ Simple and easy to understand
‚Ä¢ Works well for small datasets
‚Ä¢ Not suitable for large datasets
‚Ä¢ Number of comparisons is fixed and does not depend on input order
‚Ä¢ Performs at most (n ‚àí 1) swaps
Conclusion
Selection Sort is a Brute Force sorting method that repeatedly selects the smallest element from the unsorted list and
places it in its correct position. Although easy to implement, it is inefficient for large inputs due to its O(n¬≤) time complexity.

14.
Explain merge sort technique and find the time complexity.
Answer & Solution
Merge Sort is a divide-and-conquer sorting technique.
It works by dividing the list into smaller sublists, sorting them, and then merging the sorted sublists to produce the final sorted list.
Working Principle
Merge Sort follows three main steps:
Divide
‚Ä¢ Split the given list into two equal (or nearly equal) halves.
‚Ä¢ Continue dividing each half until each sublist contains only one element.
Conquer
‚Ä¢ Sort the smaller sublists (a single element is already sorted).
Combine (Merge)
‚Ä¢ Merge two sorted sublists into one sorted list by comparing elements one-by-one and arranging them in order.
‚Ä¢ Continue merging until the full list is sorted.
Example
Consider the list:
38, 27, 43, 3, 9, 82, 10
Step 1: Divide repeatedly
[38, 27, 43, 3, 9, 82, 10]
‚Üí [38, 27, 43, 3] and [9, 82, 10]
‚Üí [38, 27] [43, 3] [9, 82] [10]
‚Üí [38] [27] [43] [3] [9] [82] [10]
Step 2: Merge sorted parts
[27, 38] [3, 43] [9, 82] [10]
Step 3: Merge again
[3, 27, 38, 43] [9, 10, 82]
Step 4: Final merge
[3, 9, 10, 27, 38, 43, 82]
So the list is now sorted.
Time Complexity
At every level, the list is divided into two halves, and merging takes linear time.
Number of levels = log‚ÇÇn
Work done per level = n
Therefore,
Time Complexity = n log‚ÇÇn
So:
Best case: O(n log n)
Average case: O(n log n)
Worst case: O(n log n)
Space Complexity
O(n) extra space is required for merging.
Conclusion
Merge Sort is an efficient, stable, divide-and-conquer sorting algorithm with a guaranteed time complexity of O(n log n).
However, it requires additional memory for merging, which makes it less suitable for memory-restricted systems.
